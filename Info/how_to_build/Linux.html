  1. install NetBeans IDE (e.g. via package manager);
    install eclipse

  • C++ -plugin for NetBeans installieren (menu „Tools“->plugins->tab “available plugins“ name: „C/C++“
    eclipse: Help->Install new software->Programming languages->Eclipse C/C++Development tools -plugin for

  1. install the C++ -compiler (g++) (e.g. via package manager)

  2. install wxWidgets libraries  (e.g. via package manager)

  • wx>>version<<-headers

  • Desktop-specific package (e.g. for GNOME:GTK -> libwxgtk2.8-0)

  • a dir like /usr/lib/wx/include/base-unicode-release-2.8 must exist (for „#include wx/setup.h“

  1. install the Xerces libraries

    • libxerces-c-dev

    • libxerces-c3.0
  • eclipse:
    • project->properties:
      • "C/C++ Build": change tool chain editor to "Linux GCC", else warning
        "the configuration support is not installed on the system"
      • "C/C++ general" ->"Indexer":
        Set "Build configuration for the indexer" to
        "Use active build configuration" in order to use the built-in include dirs
        (verify in the  "project explorer" under "Project name->Includes")

  • Preprocessor.:

    • __WXGTK__

    • _DEBUG (wenn nicht: Laufzeitfehler, dass wxWidgets und das Programm unterschiedliche erstellt wurden)
    • COMPILE_WITH_XERCES
    • _GLIBCXX_DEBUG_ (anscheinend unnötig)
    • _UNICODE (if wxWidgets also has UNICODE->see wxUSE_UNICODE / wxchar.h)
  • Include-dirs:
    • /home/sgebauer/SourceCodeManagement/X86Info_and_Control (da sich die #include-Anweisungen in den Quelldateien auf diesen Pfad beziehen)
    • /usr/include/wx-2.8
    • /usr/lib/wx/include/gtk2-unicode-debug-2.8 (für wx/setup.h)
    • ../../SourceCodeManagement/X86Info_and_Control/Controller/MSVC_adaption
    • /usr/include (für Xerces include files (#include <xercesc/...> )
  • Linker-Libs (/usr/lib)
    • xerces-c (; statische Lib/ „*.a“): /usr/lib/libxerces-c.a
    • wx_gtk2ud_core-2.8 (.so-Datei)
    • wx_baseud-2.8 (.so-Datei)
    • wx_gtk2ud_adv-2.8 (.so-Datei)
