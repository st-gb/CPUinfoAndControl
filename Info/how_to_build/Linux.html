Build x86Info&control GUI under Linux GTK

GUI Debug-Version von X86info&ctrl erstellen für GNOME

  1. NetBeans installieren (e.g. via package manager);

  • C++ -plugin for NetBeans installieren (menu „Tools“->plugins->tab “available plugins“ name: „C/C++“

  1. C++ -compiler inst. (g++) (e.g. via package manager)

  2. wxWidgets-Libs installieren (e.g. via package manager)

  • wx>>version<<-headers

  • Desktop-specific package (e.g. for GNOME:GTK -> libwxgtk2.8-0)

  • a dir like /usr/lib/wx/include/base-unicode-release-2.8 must exist (for „#include wx/setup.h“

  1. Xerces-Libs installieren

  • libxerces-c-dev

  • libxerces-c3.0

  • Präproz.:

  • __WX_GTK__

  • _DEBUG (wenn nicht: Laufzeitfehler, dass wxWidgets und das Programm unterschiedliche erstellt wurden)

  • COMPILE_WITH_XERCES

  • _GLIBCXX_DEBUG_ (anscheinend unnötig)

  • _UNICODE (if wxWidgets also has UNICODE->see wxUSE_UNICODE / wxchar.h)

  • Include-dirs:

  • /home/sgebauer/SourceCodeManagement/X86Info_and_Control (da sich die #include-Anweisungen in den Quelldateien auf diesen Pfad beziehen)

  • /usr/include/wx-2.8

  • /usr/lib/wx/include/gtk2-unicode-debug-2.8 (für wx/setup.h)

  • ../../SourceCodeManagement/X86Info_and_Control/Controller/MSVC_adaption

  • /usr/include (für xerces)

  • Linker-Libs (/usr/lib)

  • xerces-c (; statische Lib/ „*.a“)

  • wx_gtk2ud_core-2.8 (.so-Datei)

  • wx_baseud-2.8 (.so-Datei)

  • wx_gtk2ud_adv-2.8 (.so-Datei)

  • ausführen

  • „run>run directory“: Verzeichnis, dass „configuration“-Verz beinhaltet
