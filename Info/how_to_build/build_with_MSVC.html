Build x86Info&Control with MSVC++ under Windows

  • preprocessor: "WIN32;__WXMSW__;_WINDOWS;NOPCH;LINK_TO_WINRING0_STATICALLY;XML_LIBRARY"

  • add needed Source files („.c“, ".cpp") to MSVC project

  • unter Projekt-Eigenschaften einstellen (sonst Fehler beim Erstellen)

  • C/C++ -> Codegenerierung: Laufzeitbibliothek "Multithreaded-DLL (/MD)"


-for wxWidgets:

-create x64 or 32 bit libraries if they do not exist. (choose „project->create new“ in order for the object files to be of the desired plaform (32 or 64 bit)

-add preprocessor macro: „__WXMSW__“

-bei "Linker"

-unter "Allgemein" -> Zusätzliche Bibkiotheksverzeichnisse: "\lib"-Verzeichnis von wxWidgets:

-falls Release-Version:\lib

-unter "Eingabe" hinzufügen:

  • "wxmsw28_core.lib

  • rpcrt4.lib (needed for class Uuid)

  • wxbase28.lib

  • comctl32.lib

-for Debug version:

-Rpcrt4.lib


-for xerces XML:

  • bei Linker->Eingabe hinzufügen:

  • "xerces-c_static_2.lib" (Xerces 2 linked into the x86Info&control exe)

  • "xerces-c_static_3.lib" (Xerces 3 linked into the x86Info&control exe)

  • With Xerces < Version 2.8:

  • Add "XML_LIBRARY" to "Preprocessor Definitions" to compile with Xerces statically (else many "LNK2001" and "LNK2019" and linker errors).

  • with Xerces 3.0: "XERCES_STATIC_LIBRARY" as preprocessor macro

  • add xerces include directory to C++

  • for linker: add "xerces\lib" dir

  • And: Add "Advapi32.lib" as additional dependancy for the linker.



-zu Linker->Bibliothek ignorieren hinzufuegen (sonst: "LIBCMT.lib(_ctype.obj) : error LNK2005: _isspace ist bereits in MSVCRT.lib(MSVCR80.dll) definiert.") :

  • release version: "LIBCMT.lib",

  • debug version: LIBCMTD.lib

  • from the directory from the Windows SDK for Windows Vista

  • bei Linker->Eingabe hinzufügen: "powrprof.lib"
