Usage

To find low voltages for a specific CPU core frequency:
  1. Start the GriffinControl GUI without any or little voltage / Voltage ID difference to the default voltages in the (configuration file) settings (else the Operating System may freeze without knowing the lowest stable voltage for a specific performance state) : >>example<<
  2. Disable the OS's dynamic voltage and frequency scaling:

    If the current performance state is not "0" like here:

    and you start a high load and it doesn't change to performance state "0" 
    then it's pretty sure that the OS's dynamic frequency scaling is disabled.
  3. Open the settings for the CPU core you want to test (example here: for core "0"):

  4. select the p-state you want to test (note: Windows only uses the performance states "0", "1" [and "2"] for its dynamic frequency and voltage scaling, so you usually only need to test these ones):
  5. use the option "set as current p-state after write":

    else the new voltage is not applied
  6. for the core to test: start a high load thread in order to ensure operating stability at the latest voltage:
      • e.g. use the "start high load thread" menu item:
      • or calculate a faculty with Windows' calculator
      • or run a CPU stress application like Prime95 or ORTHOS
        • then you probably have to start as many threads as there are CPU cores because else you can not assign a high load thread to a specific CPU core with Prime95
          (I think its because if there is  more than 1 CPU core then Windows' process scheduler switches the thread among the cores to achieve a similar load for each core)
        • or you start one thread and use Windows' task manager to assign the stress program to 1 CPU core:
        advantage with Prime95: often you see a calculation error when it stops calculating that is a hint that the voltage is too low before the device freezes:

        then you should increase the voltage for the performance state the calculation error occured in. So you do not need to reboot.

  7. decrease the voltage (=increase the voltage ID) by 1 step :
    hit the tabular key until the voltage ID slider has the input focus

    Now press the right arrow key once shortly so that the voltage ID is increased by 1.
  8. memorize the current "voltage IDentifier"  (let's call it "current_VID"); 
  9. finally press the "Write p-state" button:
  10. wait some seconds or minutes to check if the settings are stable
    • If the voltage does not seem to be too low, e.g.
      • not freezed
      • Prime95 has no calculation errors
      then go to step decrease the voltage by 1 step
    • If  there is a sign that the voltage is too low, e.g.:
      • device (e.g laptop) is freezed: 
        • restart the device (e.g. laptop)
        • remember the VID "current_VID" you set or start Windows in safe mode and watch the latest log data for the last set voltage
      • Prime95 has calculation errors
      then do the following:
      • use the VID "low_stable_voltage" =  "current_VID - 2" or "current_VID - 3" for the configuration (file) for the performance state / frequency that was tested.
      • if you sometimes switch from altenate current to direct current (battery mode) what is pretty common with notebooks:
        test if the "low_stable_voltage" is really stable by a harder test:
        If the CPU is inside a device that switches from AC to DC in case of  no more current from the power adapter (like usual notebooks do):
        1. insert a charged battery into the notebook
        2. put it into Alternate Current mode (so it gets its current from power adapter) 
        3. remove from Alternate Current mode (e.g unplug the power cord) so it switches to Direct Current (=battery ) mode
          • if the device is not freezed:
            go to step "put it into Alternate Current mode"  and you want to ensure that this was not just a piece of luck
          • if the device is freezed:
            "low_stable_voltage" = "low_stable_voltage -1 " or "low_stable_voltage -2 "
            go to step "put it into Alternate Current mode"
        5. now you can specify "low_stable_voltage" in the configuration file for the p-state you tested for the service and/ or GUI
